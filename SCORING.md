# Scoring & Card Selection

Документ описывает, как считается состояние слова и как выбираются слова в сессии. Все константы выведены отдельно — в одном файле конфига будет легко тюнить.

## 1. Состояние слова (что хранится)

Для каждого слова в БД храним:

| Поле | Тип | Описание |
|---|---|---|
| `created_at` | datetime | когда добавлено |
| `last_seen_at` | datetime \| null | последний показ в сессии |
| `last_correct_at` | datetime \| null | последний успешный ответ |
| `streak` | int | серия "good" подряд (сбрасывается на "again", не сбрасывается на "hard") |
| `total_good` | int | всего "угадал" (включая "сложно") |
| `total_again` | int | всего "не угадал" |
| `total_hard` | int | всего "сложно" |
| `forget_count` | int | сколько раз ошибся ПОСЛЕ того, как `streak ≥ MASTERED_THRESHOLD` |
| `history` | list[Answer] | список ответов: `{ts, grade, mode}` |

`grade ∈ {again, hard, good}`. `mode ∈ {forward, reverse, typing}`.

## 2. Что значит "выучено"

Слово считается **mastered**, когда `streak ≥ MASTERED_THRESHOLD` (по умолчанию 3). Это не финальное состояние — слово продолжает показываться, но реже, через увеличивающиеся интервалы.

Если пользователь ошибается в mastered-слове, `streak → 0` и `forget_count += 1`. После такого "падения" слово растёт по интервалам медленнее (см. п. 4).

## 3. Перевод оценки в изменение `streak`

| Кнопка | Изменение |
|---|---|
| **again** (не угадал) | `streak = 0`, `total_again += 1`, если было mastered → `forget_count += 1` |
| **hard** (сложно) | `streak += 1`, `total_hard += 1`, но интервал × 0.5 (см. ниже) |
| **good** (угадал) | `streak += 1`, `total_good += 1` |

Режим ввода с клавиатуры (см. п. 10) использует автогрейд по дистанции редактирования — подробности там.

## 4. Интервал до следующего показа

Базовый интервал по `streak`:

| streak | базовый интервал |
|---|---|
| 0 | 10 минут (та же сессия) |
| 1 | 1 день |
| 2 | 3 дня |
| 3 | 7 дней (порог mastered) |
| 4 | 21 день |
| 5 | 60 дней |
| 6+ | 180 дней |

Формула: `base_interval[min(streak, 6)]`.

**Поправки на нюансы:**

- **"hard"** → итоговый интервал = `base × 0.5` (повторим раньше).
- **`forget_count`** → штраф к росту: `effective_interval = base × decay_factor`, где
  ```
  decay_factor = 1 / (1 + forget_count × 0.4)
  ```
  То есть слово, которое забывал 1 раз → интервал × 0.71; 2 раза → × 0.56; 3 раза → × 0.45. Слова, которые ты систематически забываешь, не уходят из обоймы.

Итого:
```
interval = base_interval[min(streak, 6)] × hard_modifier × decay_factor
next_due_at = last_seen_at + interval
```

## 5. Выбор слова в сессии

В сессии для каждого кандидата считаем **вес** (положительное число), потом делаем weighted random sample без возврата.

### Режим "новые" (learn)

Сессия учит **до победного**: показываем новые слова в цикле, пока все они не достигнут `streak ≥ MASTERED_THRESHOLD`. Сессия заканчивается, когда выученных = всех новых.

**Очередь в сессии "новых"** (Leitner-style, в рамках одной сессии, без привязки к календарным интервалам):

1. На старте берём все слова с `total_good + total_again + total_hard == 0` (или ≤ N, если задан `max_new_per_session`).
2. Кладём их в очередь в случайном порядке.
3. Показываем head очереди. По ответу:
   - **good** → если `streak ≥ MASTERED_THRESHOLD` — выкидываем из очереди (выучено в этой сессии). Иначе — кладём в конец очереди.
   - **hard** → кладём не в самый конец, а на позицию `~len(queue) × 0.5` (повторим раньше).
   - **again** → кладём на позицию 2-3 (быстро вернуть, но не сразу за собой).
4. Когда очередь пуста — сессия закончена, показываем итоги.

В режиме "новые" — только forward direction (картинка → слово), без реверса.
Размер сессии (`max_new_per_session`) по умолчанию — `null` (все новые сразу).

### Режим "повтор" (review)

Кандидаты: **все** слова, у которых уже есть хотя бы один ответ.
Размер сессии: 100 слов по умолчанию (конфигурируется).
Вес для каждого:

```
weight = error_factor × due_factor × recency_dampener
```

#### `error_factor` — поощрение проблемных слов

```
attempts = total_good + total_again + total_hard
error_rate = (total_again + 0.5 × total_hard) / max(1, attempts)
prior = 0.3                  # для слов с малой статистикой
confidence = min(1, attempts / 5)
smoothed = confidence × error_rate + (1 - confidence) × prior

error_factor = 1 + 4 × smoothed
```

- Слово, в котором всегда ошибаешься → `error_factor ≈ 5`
- Слово с одной ошибкой и одним успехом → `error_factor ≈ 2.6`
- Свежее слово без статистики → `error_factor = 2.2`
- Идеально выученное слово → `error_factor → 1`

#### `due_factor` — насколько слово "просрочено"

```
t_since = now - last_seen_at         (если null → большое число)
ratio = t_since / interval

if ratio < 0.5:    due_factor = 0.05    # совсем не пора
elif ratio < 1:    due_factor = 0.3     # скоро будет пора
elif ratio < 2:    due_factor = 1.5     # пора
elif ratio < 5:    due_factor = 3.0     # давно пора
else:              due_factor = 5.0     # сильно просрочено
```

Слово, которое ты учил 1 день назад при `streak=1` (interval=1д) → `ratio≈1` → `due_factor=1.5`.
Слово, выученное месяц назад при `streak=4` (interval=21д) → `ratio≈1.4` → `due_factor=1.5`.
Слово, выученное год назад при `streak=4` → `ratio≈17` → `due_factor=5`.

#### `recency_dampener` — не показывать только что увиденное

```
minutes_since = (now - last_seen_at) / 60
if minutes_since < 2:    recency_dampener = 0      # совсем нет
elif minutes_since < 10: recency_dampener = 0.2    # маловероятно
else:                    recency_dampener = 1
```

Это нужно, чтобы внутри одной сессии слово не выпадало подряд два раза.

### Сэмплинг

`numpy.random.choice(words, p=weights / sum(weights))` для каждого слота сессии. Размер сессии задаётся юзером (напр. 20). Если в режиме "новые" слов меньше, чем размер сессии — берём сколько есть.

## 6. Направление показа (forward/reverse)

В сессии "повтор" каждое слово случайно показывается в одном из двух направлений (50/50, конфигурируемо):

- **forward**: картинка → угадать сербское слово
- **reverse**: сербское слово → вспомнить значение, переворот → картинка для проверки

Статистика по направлениям копится отдельно в `history`, но `streak` общий (это одно знание слова с двух сторон).

В сессии "новые" — только forward, чтобы не перегружать.

## 7. История сессий (отдельно от слов)

Каждая сессия пишет в `sessions.json` запись:

```json
{
  "id": "uuid",
  "started_at": "...",
  "ended_at": "...",
  "mode": "review" | "learn",
  "results": [
    {"word_id": "...", "grade": "good", "direction": "forward"}
  ],
  "summary": {
    "shown": 20, "good": 15, "hard": 3, "again": 2,
    "accuracy": 0.75, "new_mastered": 1
  }
}
```

В углу экрана во время сессии: `15/2 ✓✗ · acc 88%` (хорошие / плохие / точность).

## 8. Конфиг (`config.json`)

Все магические числа собираем здесь. Большинство выведено в UI настроек как ползунки/чекбоксы.

```json
{
  "mastered_threshold": 3,
  "base_intervals_minutes": [10, 1440, 4320, 10080, 30240, 86400, 259200],
  "hard_modifier": 0.5,
  "forget_decay_alpha": 0.4,
  "error_factor_alpha": 4,
  "error_prior": 0.3,
  "review_session_size": 100,
  "max_new_per_session": null,
  "reverse_probability": 0.5,
  "due_thresholds": [0.5, 1, 2, 5],
  "due_factors": [0.05, 0.3, 1.5, 3.0, 5.0],
  "typing_mode_enabled": false,
  "typing_relaxed_diacritics": true,
  "typing_hard_levenshtein_threshold": 2,
  "always_regenerate_example": false
}
```

В UI (крутилки в настройках):
- `error_factor_alpha` — "насколько агрессивно показывать проблемные слова" (1–10)
- `mastered_threshold` — "сколько правильных подряд для запоминания" (1–7)
- `review_session_size` — размер сессии повтора (10–500)
- `reverse_probability` — % обратных карточек в сессии (0–100)
- `typing_mode_enabled`, `typing_relaxed_diacritics`, `always_regenerate_example` — чекбоксы.

Меняется руками без перезапуска (читаем при старте каждой сессии).

## 10. Режим ввода с клавиатуры (typing mode)

Включается опцией `typing_mode_enabled`. Когда включён — в forward-направлении вместо самооценки нужно набрать слово на клавиатуре.

**Нормализация при сравнении:**

И введённая строка, и эталон проходят через одну функцию `normalize()`:

1. Lowercase, trim.
2. Если включено `typing_relaxed_diacritics` (по умолчанию **true**):
   - Снимаем диакритику: `š→s, č→c, ć→c, ž→z, đ→dj`.
   - Кириллицу транслитерируем в латиницу: `ћ→c, ђ→dj, ј→j, љ→lj, њ→nj, џ→dz, ш→s, ч→c, ж→z, ...` (полная таблица соответствий).
   - В эталоне применяем то же самое.
3. Дистанция редактирования (Levenshtein) между нормализованными строками.

**Грейдинг:**

| дистанция | оценка |
|---|---|
| 0 | **good** |
| 1–2 | **hard** (показать "почти: правильно было *X*") |
| ≥3 | **again** |

Порог `typing_hard_levenshtein_threshold = 2` конфигурируется.

**Релевантно только для forward**. В reverse направлении (слово → картинка) ввод не имеет смысла — там самооценка по картинке.

## 11. Что НЕ делаем (намеренно)

- Не реализуем полноценный SM-2 / FSRS — слишком сложно, мало данных.
- Не подбираем интервалы по EF (easiness factor) — `forget_count` достаточно.
- Не учитываем время суток / "энергию пользователя" — переусложнение.
- Не используем нейросети для предсказания "забуду / не забуду".

Если через месяц использования окажется, что веса плохие — тюним константы в config.json, а не переписываем алгоритм.
